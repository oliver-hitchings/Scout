# Scout 0.1.0-beta.12

Scout beta.12 prevents repeated vacancies from cluttering the tracker and adds
safe, visible application-update checks while retaining the Beta 11 VPS CSP fix.

## Highlights

- Checks new jobs against both the current scan and the existing tracker before
  saving them, using provider IDs, canonical URLs and normalised role evidence.
- Collapses the same underlying vacancy across providers, retains every source,
  and preserves status, notes, applications, contacts, company history and chats.
- Treats a materially changed repost as an advert update to review while keeping
  genuinely different openings at the same company separate.
- Shows an in-app notification for newer official Scout releases, including the
  version and a link to the published release notes.
- Offers an explicit opt-in to download the correct Windows, macOS or Linux
  package automatically; notification-only remains the default.
- Verifies downloads against the release SHA-256 manifest, rejects unsafe asset
  names and stores packages in device-local state outside the private workspace.
- Keeps installation user-confirmed and restricts host update administration to
  the local machine. Remote owner clients can continue using Scout and see update
  notices without being able to change the host.
- Carries forward the Beta 11 strict-CSP UI hotfix and refreshes the service-worker
  shell so upgraded desktop and VPS clients do not retain stale actions.
- Retains the protected, rollback-capable VPS release deployment that refuses
  dirty or unexpected checkouts and preserves the workspace, provider sign-ins
  and Tailscale Serve mapping.

There is no tracker or workspace schema migration. Existing duplicate tracker
entries are not destructively rewritten; deduplication applies as jobs are
discovered or rediscovered. Scout remains local-first and never applies, sends
outreach, or installs an application update without the user's action.
