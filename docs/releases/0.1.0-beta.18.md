# Scout 0.1.0-beta.18

Beta 18 restores the private remote interface and includes the beta.17 follow-up improvements for scan visibility, reliable PDF rendering, and native macOS launch behaviour.

## Changes

- Keeps completed scan outcomes visible, including reviewed and kept totals, discard categories, source health, report access, and a bounded candidate-level rejection audit.
- Shows elapsed scan time and an approximate remaining range based on comparable local history.
- Runs one safely broadened discovery pass after a successful manual primary scan finds no keepers. Scheduled scans do not loop, and approved salary, location, commute, exclusion, evidence, and scoring gates remain unchanged.
- Renders approved master and tailored CV PDFs as bounded background operations, ties each PDF to its source hash, and replaces completed output atomically.
- Allows Scout's same-origin PDF preview while retaining strict framing protection for other responses.
- Replaces the macOS shell launcher with a native AppKit launcher that reconnects safely, avoids duplicate servers, records diagnostics, and reports startup failures.
- Restores remote-owner setup and settings access when encrypted backup is enabled. Emergency recovery-key retrieval remains restricted to the Scout host and is no longer attempted remotely.
- Expands browser coverage for the remote setup regression and scan, PDF, and launcher behaviour.

## Upgrade and compatibility

- No workspace schema migration, provider sign-in, schedule change, or backup-format change is required.
- Existing scan records remain readable; the richer scan schema is additive.
- Existing PDFs are considered stale until rendered against the current source metadata.
- Remote recovery-key handling remains host-only. Authenticated remote owners can review and use the rest of Scout normally.

## Verification and package trust

The release candidate is accepted only after the complete unit/integration suite, Chromium and Firefox browser acceptance, managed Typst compilation, privacy staging audit, native Windows/macOS/Linux package smoke tests, protected VPS deployment, and live service checks pass.

Beta packages remain unsigned. Verify the downloaded file against the published SHA-256 checksum; the checksum protects transfer integrity but does not prove publisher identity.
