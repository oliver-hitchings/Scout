# Scout 0.1.0-beta.14

Beta 14 separates everyday settings from first-run onboarding and makes long-lived installed pages recover safely after an update.

## Highlights

- The header backup status opens dedicated backup details instead of the setup wizard.
- Established workspaces use a sectioned Settings hub; search retuning is an explicit, separate flow.
- Settings and backup views keep an accessible close control, Escape/backdrop dismissal and focus restoration.
- Scout detects when its server is running a newer interface and offers a safe, user-confirmed refresh without discarding active work.
- Static shell caches use an exact build identifier, network-first navigation and browser-level regression coverage.

## Upgrade note

A page opened before this build cannot receive the new build-handshake code retroactively. Refresh that page once after upgrading. Scout detects subsequent interface updates and displays a refresh action automatically.
