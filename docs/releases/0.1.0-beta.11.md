# Scout 0.1.0-beta.11

Scout beta.11 adds company relationship history, dedicated interview preparation and optional owner-only remote access without replacing role-specific AI conversations or the local-first workspace.

## Highlights

- Groups every tracked vacancy for the same company in one relationship drawer.
- Records real correspondence and calls with a date, contact, direction, channel, type, linked role and verbatim message or notes.
- Reuses contacts by name and keeps their role and LinkedIn URL available across vacancies at the company.
- Includes existing opportunity contacts, application stages and event logs in the company timeline without rewriting older tracker data.
- Keeps each Claude or Codex conversation attached to its individual vacancy and links those job chats from the company view.
- Stores relationship records locally under `data/companies/`, inside the private Scout workspace and optional encrypted backup boundary.
- Supports removing manually recorded updates while preserving derived tracker history.
- Adds a separate Claude or Codex interview-prep conversation for every opportunity, available before or after an interview stage is recorded.
- Builds opportunity-specific prep packs with current cited company research, evidence-backed talking points, questions to ask and interactive mock interviews while preserving private notes.
- Optionally exposes the same running Scout host to the configured owner through Tailscale Serve HTTPS; Scout remains loopback-bound and never enables public Funnel access.
- Adds owner identity enforcement, an installable phone shell, supervised Windows startup and runtime recovery without copying provider credentials or deleting unrelated Tailscale mappings.
- Documents a supported single-owner Ubuntu VPS mode with conservative sizing, loopback/Tailscale isolation and a reboot-safe unprivileged user service.
- Uses the Scout icon for the Windows setup executable.

There is no tracker schema migration. Company records and interview-prep packs are created lazily, existing workspaces remain valid, and the opportunity tracker is not rewritten. Remote hosting is disabled by default and requires explicit local setup. Scout remains local-first and never sends correspondence or submits applications.
