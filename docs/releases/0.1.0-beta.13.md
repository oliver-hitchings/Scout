# Scout 0.1.0-beta.13

Beta 13 completes the private VPS operating path and makes daily reports easier to review.

## Highlights

- Supports multiple named daily jobs, including a Claude primary scan at 07:30 and a Codex second pass at 08:30.
- Migrates workspaces to schema 2 with a pre-migration configuration backup.
- Combines same-day provider runs into one Markdown report without losing the earlier run.
- Adds a responsive semantic report view with safe Markdown links, lists and legacy fallback.
- Adds strict GitHub SSH backup support for a repository-scoped VPS deploy key while retaining desktop HTTPS/Git Credential Manager support.
- Reports individual timer state, next/last runs, backup failures and last successful sync.

## Upgrade and manual actions

The VPS owner must add the generated public deploy key to the private workspace repository with write access, save the recovery key away from the VPS, enable lingering for the Scout account, and explicitly enable both timers after a healthy supervised scan.

Older Scout versions cannot open schema-2 workspaces. Install Beta 13 or later instead of lowering `schemaVersion`.
