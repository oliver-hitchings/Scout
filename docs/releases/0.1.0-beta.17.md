# Scout 0.1.0-beta.17

Beta 17 makes setup and daily use clearer, adds independent model choices for Codex and Claude, and hardens Scout against interrupted writes, overlapping work, upgrades, and recovery failures.

## Changes

- Shows persistent, reviewable progress for onboarding proposals and scans, recovers transient setup failures in place, and explains strict scans that correctly keep no opportunities.
- Adds independent Codex and Claude model settings for scheduled scans and individual job questions. Leaving a model blank continues to use that provider's default.
- Keeps provider detection off the HTTP request path by sharing asynchronous, cached probes, so a slow or unavailable CLI does not freeze unrelated Scout requests.
- Adds shared accessible dialog behaviour: initial focus, keyboard trapping, inert backgrounds, defined Escape handling, focus restoration, and nested setup-adjacent panels.
- Coordinates UI tracker changes with scans, detects stale revisions, retries safe conflicts once, and preserves notes, status, contacts, categories, commute details, and application stages.
- Makes every opportunity-table column reachable on a phone, keeps strong-match controls on screen, and preserves caret position and IME composition while filtering.
- Corrects visible setup text encoding and expands regression checks across user-facing source files.
- Decodes request bodies with bounded byte accounting and strict streaming UTF-8 validation.
- Streams update packages to restricted temporary files while incrementally hashing and enforcing the download limit, instead of buffering the complete package in memory.
- Defers interface refreshes while CV edits, chats, scans, setup work, or settings are active; remote restarts now require confirmation and refuse to interrupt active work.
- Validates restores before and after activation and rolls back even when the original target was empty.
- Standardises critical mutable workspace records on flushed, same-directory temporary files followed by atomic replacement.
- Expands browser acceptance from Ubuntu Chromium to Ubuntu Firefox and Windows Chromium, alongside the existing four-platform unit, audit, Typst, staging, and packaging gates.
- Adds persistent public operations context and documentation-maintenance rules, and keeps fork pull-request privacy audits independent of private release markers.

## Upgrade and compatibility

- No workspace schema migration or credential change is required.
- Existing provider defaults continue to work. Model selection is optional and can be changed in Settings.
- Finish or save active work before refreshing or restarting Scout after installation; the interface now explains any blocker instead of discarding that work.
- Remote hosting, private-workspace boundaries, schedules, and encrypted-backup formats are unchanged.

## Verification and package trust

The release candidate is accepted only after the complete unit/integration suite, Chromium and Firefox browser acceptance, managed Typst compilation, privacy staging audit, native Windows/macOS/Linux package smoke tests, rollback rehearsal, successful private VPS rehearsal, and live service checks pass.

Beta packages remain unsigned. Verify the downloaded file against the published SHA-256 checksum; the checksum protects transfer integrity but does not prove publisher identity.
