# Scout 0.1.0-beta.10

Scout beta.10 repairs the Windows acceptance blockers and replaces provider-owned workspace editing with bounded, runtime-owned workflows.

## Highlights

- Detects official per-user Codex and Claude Windows installations and distinguishes absent, signed-out, authenticated and authenticated-but-outdated CLIs.
- Generates evidence-traceable onboarding proposals in one no-tools provider turn; Scout stages, reviews, atomically activates and rolls back all five files itself.
- Keeps scan locks, source collection, artifact writing and validation inside Scout's trusted runtime, independent of Git availability.
- Caps source candidates and provider context, skips AI for zero-result scans, reports healthy-empty/degraded/failed outcomes truthfully and validates artifacts before success.
- Retries hiring.cafe transient failures and refreshes its build ID once after endpoint changes.
- Enforces hard exclusions and employer-declared mandatory requirements before recomputing trusted scores and action bands.
- Adds a bounded provider-identified **Fit and evidence gaps** chat action with authoritative selected-job context and private transcript persistence.
- Records only genuine write/edit events as touched files; reads, searches and fetches remain visible activity without false file-change claims.
- Preserves selectable-text PDF import on Node 24 while continuing to reject scanned/image-only PDFs.
- Corrects the no-schedule Windows quit wording and shuts down the trusted runtime explicitly.

There is no workspace schema migration or new external service. Existing tracker fields and user-authored history remain compatible. Scout remains unsigned, local-first, never applies or sends outreach, and sends only bounded task context to the provider account the user selects.
