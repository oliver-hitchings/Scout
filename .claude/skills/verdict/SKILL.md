# Turn reactions into calibration

1. Resolve the tracker item and read its score plus `profile/calibration.md`.
2. Append the reaction verbatim with today's date and update status only when implied.
3. Record confirmed progress using the canonical tracker event values.
4. Append a dated calibration precedent only when the reaction generalises to future opportunities; never rewrite older precedents.
5. Re-score only when supported, explain old versus new, and report other possibly affected items without bulk-changing them.
6. Commit with `verdict: <company> - <gist>`.

Preserve all earlier history.
